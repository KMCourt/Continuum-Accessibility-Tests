🧰 How to Test — Tools Guide

▼
📱 Real Device Testing

Test on a physical iPhone or Android device for the most accurate results.

iOS VoiceOver: Settings → Accessibility → VoiceOver. Swipe to navigate, double-tap to activate.

Android TalkBack: Settings → Accessibility → TalkBack. Swipe right/left to navigate.

🖥️ Chrome DevTools Emulation

Open Chrome DevTools (F12) → click the device toolbar icon (Ctrl+Shift+M) → select a device from the dropdown.

Test at both portrait and landscape orientations using the rotate button.

👆 Touch Target Testing

Tap every button, link, and interactive element. Targets should be at least 44×44 CSS pixels — easy to tap without accidentally hitting adjacent elements.

Check with one finger on a real device or use Chrome's touch simulation.

🔎 Zoom & Text Size

Use pinch-to-zoom on the page — content should reflow and remain usable.

Increase system text size: iOS Settings → Display & Brightness → Text Size. Android: Settings → Display → Font Size.

🔄 Orientation Testing

Rotate the device between portrait and landscape. All content and functionality must be available in both orientations.

Check that the page does not force a specific orientation via CSS or viewport settings.

⌨️ External Keyboard

Connect a Bluetooth keyboard to a mobile device. All interactive elements should be reachable and activatable via keyboard — the same as desktop.
