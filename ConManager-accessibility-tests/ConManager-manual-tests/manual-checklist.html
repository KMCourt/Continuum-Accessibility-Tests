🧰 How to Test — Tools Guide

▼
🔊 Screen Reader

NVDA (free, Windows) — download from nvaccess.org.
Start: Win+Ctrl+Enter or open from Start Menu.

Navigate: Tab (interactive elements), Arrow keys (read content), H (headings), B (buttons), F (form fields).

On Mac/iOS: VoiceOver is built-in — Cmd+F5 on Mac, triple-press Home on iOS.

⌨️ Keyboard Navigation

No tools needed. Put the mouse aside.

Use: Tab (next element), Shift+Tab (previous), Enter/Space (activate), Arrow keys (menus/dropdowns), Escape (close modals).

Start from the browser address bar and Tab into the page.

🎨 Colour Contrast

Use the free Colour Contrast Analyser app — download from tpgi.com/color-contrast-checker — to eyedropper colours from the screen.

Also in Chrome DevTools: right-click any text element → Inspect → click the colour swatch in the CSS panel to see the contrast ratio.

🔍 Browser Extensions

Install axe DevTools (free version by Deque) from the Chrome/Firefox extension store. Open DevTools (F12) → axe DevTools tab → Scan page.

Also useful: WAVE extension from WebAIM (wave.webaim.org) — adds visual overlays showing errors directly on the page.

📱 Mobile Emulation

In Chrome: F12 → click the device toolbar icon (or Ctrl+Shift+M) → select a device (e.g. iPhone 15 Pro, Galaxy S24).

For real-device screen reader testing: VoiceOver on iOS (Settings → Accessibility → VoiceOver) or TalkBack on Android (Settings → Accessibility → TalkBack).

🔎 Zoom Testing

In any browser: Ctrl+Plus (Cmd+Plus on Mac) to zoom to 200%. Check nothing overlaps or disappears.

Also test OS-level text size: Windows Settings → Accessibility → Text size.
